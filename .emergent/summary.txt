<original_problem_statement>
The user wants to build a mobile application called ShiftExtra using React Native and Expo.

Initial requirements:
- User login system.
- Monthly calendar to log shifts (normal, extra, vacation, day off) with start/end times.
- A system to log gratuities/bonuses with date, type, value, and an optional note, associable with a shift.
- Automatic calculations for monthly and annual extra totals.
- Statistics page with monthly graphs and comparisons.
- Simple UI with dark mode and bottom tab navigation.

Expanded requirements added during development:
- The app should be a professional tool for police officers.
- **Police Incident Management:** A new section to create detailed incident reports.
    - Fields: Date, location, description, crime type.
    - Ability to add individuals (suspects, witnesses, victims) with detailed personal and identification information (Passport, Citizen Card, etc.).
    - Ability to attach photos to the incident report.
    - Generate a PDF of the completed incident report.
- **Advanced Calendar Features:**
    - Display the shift type (e.g., 'M' for Morning) with a specific color directly on the calendar day.
    - A quick select mode to pick a shift type and then tap multiple days to assign it.
    - An improved shift cycle system: select a cycle pattern, tap a start day, and it populates the rest of the month.
    - Integration of Portuguese national holidays into the calendar.
- **Bank of Hours System:**
    - Track a balance of excess hours.
    - A new shift type Excesso to deduct hours from this bank.
    - A dedicated dashboard to view the hour bank balance, total days off, and accumulated extra earnings.
- **UI/Stats Simplification:**
    - Remove annual projection and average per extra from statistics.
    - Indicate on the calendar which days have an extra/gratuity logged.
</original_problem_statement>
